The Problem

Traditional SAST was built for web apps.
Agents have new attack surfaces.

AI agents introduce attack patterns that Semgrep, Bandit, and CodeQL were never designed to detect. These tools parse syntax trees. They don't understand agentic data flow -- how user input travels through tool calls, how LLM outputs become executable code, or how memory stores get poisoned.

How It Works

Four-stage review pipeline.

S

Static Scanner

22 deterministic patterns. Zero AI cost. Catches eval chains, pickle loads, noqa abuse, unbounded loops.

→
C

Council

5 specialist AI workers in parallel. $0.001 each. Security, architecture, data flow, prompt safety, dependency.

→
N

Sentinel

Dedup and normalization. Merges overlapping findings. Assigns severity. Removes false positives.

→
D

Darwin

Per-repo learning via .lore.yml. Suppressions are git-committable. Enterprise audit trail built in.

Features

Built for CI pipelines and security teams.

--mode security

Agent-specific patterns only. Low false positive rate. Skips style and formatting noise.

--output sarif

GitHub Code Scanning native integration. Findings appear inline in the Security tab.

--strict

.lore.yml only, no auto-learn. Hard CI gating. If Darwin hasn't seen it, the build fails.

--scaffold

Auto-fix suggestions per finding. Generates patch-ready remediation code.

GitHub Actions

Inline PR comments with severity badges. REQUEST_CHANGES blocks merge on critical findings.
